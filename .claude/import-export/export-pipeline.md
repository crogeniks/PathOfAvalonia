# Export Pipeline — Extracting Game Data

The `src/Export/` tooling extracts game data from GGG's .ggpk / bundle archive into the lua tables that ship inside `src/Data/`. Runs offline against an installed PoE client.

## GGPK Parser (`Export/ggpk/`)

- `GGPKData.lua` — wraps the `bun_extract_file.exe` CLI (bundled tool) to pull a named file from the installed `.ggpk` or bundle.
- Output: raw bytes written to a temp location, then read by the extractor.

## DAT Tables

Every structured table (items, skills, passives, monsters, …) ships in GGG's `.dat` format (fixed-width rows + variable-length data).

- `DatFile.lua` — 32-bit layout reader.
- `Dat64File.lua` — 64-bit layout reader (newer PoE versions).
- Typed columns: `Int`, `Short`, `String`, `Boolean`, `Float`, `Enum`, `Key` (FK to other DAT), `KeyList`, `Array`.
- `spec.lua` (~14K lines) — the full DAT schema; enumerates every table and column.

## Per-Domain Scripts (`Export/Scripts/` + subfolders)

| Target | Files |
|--------|-------|
| Item bases | `bases.lua` + `Bases/*.lua` per slot |
| Skills / gems | `skills.lua` + `Skills/*.lua` (per attribute/category) |
| Passives | `passives.lua` + `psg.lua` (tree sprite groups) |
| Tree | `Tree/*.lua` — emit the `TreeData/<version>/tree.lua` format |
| Uniques | `Uniques/*.lua` per slot |
| Enemies | `Enemies/*.lua` |
| Minions | `Minions/*.lua` |
| Classes | `Classes/*.lua` |
| Stat descriptions | `statdesc.lua` (parses GGG's stat description .txt files) |
| Essences | `essenceMatch.lua`, `essence.txt` |

## Entry Points

- `Export/Launch.lua` — standalone entry; loaded via `HeadlessWrapper` or the regular launcher.
- `Export/Main.lua` — UI / CLI orchestration for the export steps.
- `Export/browse.lua` — interactive GGPK browser to audit files before extraction.

## Output Contract

Scripts write lua files into `src/Data/` (overwriting previous versions). The production app consumes these files verbatim; they carry `-- This file is automatically generated, do not edit!` headers.
